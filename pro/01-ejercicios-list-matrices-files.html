NRO. PRÁCTICA: 4 TÍTULO PRÁCTICA: Procesamiento de listas y matrices con funciones en Python
OBJETIVO
  • Aplicar listas y matrices en la resolución de problemas básicos de programación.
  • Organizar un programa en varios archivos usando módulos de Python.
  • Implementar funciones que procesen datos y retornen resultados mediante return.
  • Separar la lógica de cálculo, datos y presentación de resultados.
  • Ejecutar el programa desde un archivo principal sin solicitar datos por teclado.
INSTRUCCIONES:
  1. Desarrollar una aplicación en Python que procese listas y matrices usando funciones con return. El programa debe ejecutarse directamente desde main.py, sin usar input ni menú.
  2. Estructura del proyecto

    Crear una carpeta llamada practica_listas_matrices y organizar el proyecto con los siguientes archivos:

    practica_listas_matrices/
│
├── main.py
├── datos.py
├── operaciones_listas.py
├── operaciones_matrices.py
└── reportes.py
  3. Archivos y responsabilidades

    Cada archivo debe cumplir una responsabilidad específica dentro del programa.

    Archivo Responsabilidad
    datos.py Definir funciones que retornen listas y matrices base.
    operaciones_listas.py Implementar funciones para procesar listas.
    operaciones_matrices.py Implementar funciones para procesar matrices.
    reportes.py Implementar funciones encargadas únicamente de imprimir resultados.
    main.py Importar módulos, llamar funciones y ejecutar el flujo principal.
  4. Nombres obligatorios de archivos y funciones

    Los nombres deben respetarse exactamente para mantener una estructura clara y evaluable.

    Archivo Funciones obligatorias
    datos.py get_notas(), get_productos(), get_ventas(), get_matriz_numeros()
    operaciones_listas.py calcular_promedio(lista), contar_aprobados(lista), obtener_mayor(lista), contar_productos_largos(productos), buscar_producto(productos, nombre)
    operaciones_matrices.py sumar_matriz(matriz), mayor_matriz(matriz), sumar_filas(matriz), sumar_columnas(matriz), contar_pares_matriz(matriz)
    reportes.py mostrar_reporte_notas(...), mostrar_reporte_productos(...), mostrar_reporte_matriz(...)
  5. Ejecución desde archivo principal

    El programa debe ejecutarse desde main.py. Este archivo no debe contener toda la lógica del programa; solo debe obtener datos, llamar funciones y mostrar resultados.

  6. Restricciones
    • No usar input.
    • No usar menú.
    • No usar librerías externas.
    • No usar sum, max, min ni funciones automáticas para resolver directamente la lógica principal.
    • No colocar toda la solución dentro de main.py.
    • No imprimir dentro de las funciones de operaciones.
    • Las funciones de operaciones deben calcular y retornar resultados.
  7. Entregables

    Subir al AVAC todos los archivos desarrollados durante la práctica.

    • main.py
    • datos.py
    • operaciones_listas.py
    • operaciones_matrices.py
    • reportes.py

    Los archivos pueden subirse individualmente o dentro de una carpeta comprimida llamada practica_listas_matrices.

  8. Criterio de no calificación
    • No se calificará si las funciones de operaciones imprimen en lugar de retornar resultados.
    • No se calificará si el programa no ejecuta correctamente desde main.py.
