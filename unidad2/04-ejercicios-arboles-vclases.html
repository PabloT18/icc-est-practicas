NRO. PRÁCTICA: 4 TÍTULO PRÁCTICA: Ejercicios de lógica con estructuras no lineales: árboles
OBJETIVO
  • Mejorar la comprensión de lógica de programación usando estructuras de datos no lineales.
  • Aplicar árboles binarios y árboles binarios de búsqueda en la resolución de problemas lógicos.
  • Implementar algoritmos de inserción, inversión, recorrido por niveles y cálculo de profundidad.
  • Organizar el código en carpetas, clases y métodos según una estructura definida.
INSTRUCCIONES:
  1. Leer y comprender el requerimiento

    Antes de programar, el estudiante debe leer detenidamente el enunciado de cada ejercicio para identificar la entrada, el proceso esperado y la salida que debe producir el algoritmo.

  2. Implementar la solución

    Desarrollar el código necesario para resolver cada uno de los ejercicios propuestos usando árboles binarios y árboles binarios de búsqueda.

  3. Organización de los ejercicios

    Crear una carpeta para cada ejercicio:

    • Ejercicio_01_insert: insertar valores en un árbol binario de búsqueda.
    • Ejercicio_02_invert: invertir un árbol binario.
    • Ejercicio_03_listLeves: listar los niveles del árbol en listas enlazadas.
    • Ejercicio_04_depth: calcular la profundidad máxima del árbol.
  4. Estructura del proyecto

    La estructura de directorios del proyecto debe quedar de la siguiente forma:

    src/

  5. Nombres obligatorios de clases y métodos

    Los nombres deben respetarse exactamente para que las pruebas unitarias puedan ejecutarse correctamente.

    Probar el código con diferentes casos de prueba para asegurar que los algoritmos funcionan correctamente con árboles vacíos, árboles de un solo nodo y árboles con varios niveles.

  7. Repositorio

    Al finalizar la práctica, el estudiante debe realizar commit y push al repositorio.

    Mensaje de commit: Estructuras No Lineales – Ejercicios Arboles

    El README debe incluir la descripción de esta versión e indicar cómo sirve el código desarrollado para cada ejercicio.

ACTIVIDADES POR DESARROLLAR

1. Crear la estructura base del proyecto

Crear las carpetas y clases indicadas. La clase Node debe permitir representar un nodo de árbol binario con valor entero, referencia izquierda y referencia derecha.

public class Node {
    int value;
    Node left;
    Node right;

    public Node(int value) {
        this.value = value;
        this.left = null;
        this.right = null;
    }
}

2. Ejercicio 01: Insertar en un Árbol Binario de Búsqueda

Carpeta: Ejercicio_01_insert

Implementar un algoritmo para insertar valores en un árbol binario de búsqueda. Un BST es un árbol binario donde cada nodo cumple las siguientes reglas:

  • El subárbol izquierdo de un nodo contiene solo valores menores que el nodo.
  • El subárbol derecho de un nodo contiene solo valores mayores que el nodo.

Clase requerida:

public class InsertBSTTest {

    public Node insert(int[] numeros) {
        // add code
    }
}

Ejemplo:

Input: [5, 3, 7, 2, 4, 6, 8]

Output:
      5
   3     7
 2  4  6  8

CONCLUSIONES:
  • Aplicación práctica de estructuras de datos: esta práctica permite aplicar los conceptos teóricos de árboles en problemas prácticos mediante inserción, inversión, recorrido por niveles y cálculo de profundidad.
