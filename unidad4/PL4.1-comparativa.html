NRO. PRÁCTICA: 4 TÍTULO PRÁCTICA: Comparación situacional de algoritmos de ordenamiento
OBJETIVO (Colocar el o los objetivos que se alcanzarán al desarrollar la práctica):
  • Implementar algoritmos de ordenamiento sobre arreglos de objetos en Java.
  • Comparar el comportamiento de Inserción y QuickSort en arreglos desordenados y casi ordenados.
  • Medir tiempos de ejecución utilizando una clase de benchmarking propia.
  • Analizar cómo el estado inicial de los datos influye en el rendimiento de un algoritmo.
INSTRUCCIONES:
  1. Desarrollar una aplicación en Java que permita generar arreglos de objetos de tipo Persona y ordenarlos utilizando únicamente los siguientes algoritmos:
    • Inserción.
    • QuickSort.
  2. Modelo de datos

    Se debe crear una clase Persona con los siguientes atributos:

    • String nombre
    • int edad

    Además, la clase debe tener un método calculado que permita comparar dos personas. El ordenamiento se realizará primero por edad y, si dos personas tienen la misma edad, se usará el valor numérico generado a partir del nombre.

    Ejemplo del criterio de comparación:

    public int getCriterioOrdenamiento() {
    int valorNombre = 0;

    for (int i = 0; i < nombre.length(); i++) {
        valorNombre += nombre.charAt(i);
    }

    return edad * 100000 + valorNombre;
}

    El método anterior permite obtener un número que combina la edad y el nombre. Este valor será usado por los algoritmos para ordenar el arreglo de personas.

  3. Generación de datos

    El programa debe incluir un método generador que reciba como parámetro el tamaño del arreglo y retorne un arreglo de personas.

    El método debe cumplir lo siguiente:

    • Generar nombres con el formato Persona 1, Persona 2, Persona 3, etc.
    • Generar edades aleatorias entre 0 y 100 años.
    • Permitir crear arreglos de 10.000, 50.000 y 100.000 personas.
    public static Persona[] generarPersonas(int cantidad) {
    Persona[] personas = new Persona[cantidad];

    for (int i = 0; i < cantidad; i++) {
        String nombre = "Persona " + (i + 1);
        int edad = (int) (Math.random() * 101);
        personas[i] = new Persona(nombre, edad);
    }

    return personas;
}
  4. Clase Benchmarking

    Se debe crear una clase Benchmarking que permita medir el tiempo de ejecución de cualquier algoritmo enviado como función.

    La clase debe usar System.nanoTime() para calcular la duración de cada ejecución.

    Debe recibir:

    • La función que ejecuta el algoritmo.
    • El nombre del algoritmo.
    • El tamaño de la muestra.
    • El escenario evaluado.

    Debe retornar un objeto Resultado con el nombre del algoritmo, tamaño de muestra, escenario y tiempo obtenido.

  5. Clase Resultado

    Se debe crear una clase Resultado para almacenar los datos de cada medición.

    Debe contener al menos:

    • String algoritmo
    • String escenario
    • int sample
    • double tiempoNano
    • double tiempoMilis
  6. Clase de control de ordenamiento

    Se debe crear una clase controladora, por ejemplo SortPersonaMethods, que implemente los métodos:

    • insertionSort(Persona[] personas)
    • quickSort(Persona[] personas, int inicio, int fin)

    No se permite utilizar métodos de ordenamiento propios de Java, como Arrays.sort(), Collections.sort() o similares.

  7. Uso correcto de copias

    Cada algoritmo debe trabajar con una copia independiente del arreglo original. No se debe medir un algoritmo sobre un arreglo que ya fue modificado por otro algoritmo.

    Persona[] base = generarPersonas(size);
Persona[] copiaInsercion = base.clone();
Persona[] copiaQuickSort = base.clone();

    Si no se realizan copias independientes, la comparación de tiempos no será válida.

4. Crear la clase Benchmarking

La clase Benchmarking debe medir el tiempo de ejecución de una función.

package utils;

import java.util.concurrent.Callable;
import models.Resultado;

public class Benchmarking {
    public static Resultado medirTiempo(
            Callable<Void> funcion,
            String algoritmo,
            String escenario,
            int sample
    ) {
        try {
            long inicio = System.nanoTime();
            funcion.call();
            long fin = System.nanoTime();

            double duracion = fin - inicio;

            return new Resultado(algoritmo, escenario, sample, duracion);
        } catch (Exception e) {
            throw new RuntimeException("Error al ejecutar la función de ordenamiento");
        }
    }
}

5. Crear los métodos de ordenamiento

La clase SortPersonaMethods debe implementar Inserción y QuickSort.

package controllers;

import models.Persona;

public class SortPersonaMethods {

    public void insertionSort(Persona[] personas) { 
        //add code
    }

    public void quickSort(Persona[] personas, int inicio, int fin) {
        if (inicio < fin) {
            int indicePivote = particionar(personas, inicio, fin);
            quickSort(personas, inicio, indicePivote - 1);
            quickSort(personas, indicePivote + 1, fin);
        }
    }

    private int particionar(Persona[] personas, int inicio, int fin) {
        Persona pivote = personas[fin];
        int i = inicio - 1;

        for (int j = inicio; j < fin; j++) {
            if (personas[j].getCriterioOrdenamiento() <= pivote.getCriterioOrdenamiento()) {
                i++;
                intercambiar(personas, i, j);
            }
        }

        intercambiar(personas, i + 1, fin);
        return i + 1;
    }

    private void intercambiar(Persona[] personas, int i, int j) {
        Persona aux = personas[i];
        personas[i] = personas[j];
        personas[j] = aux;
    }
}

  • 50.000 personas.
  • 100.000 personas.
